Basic Syntax

The fundamental rules of a computer language are represented by basic syntax. It is impossible to create functional code without these rules. Each language's fundamental syntax is composed of a unique collection of rules.

  • PHP Scripts can be placed anywhere in the document.
  • A PHP Script starts with "<?php", and ends in "?>".
  • A PHP file's extension is ".php"
  • A PHP file normally contains HTML tags, and some PHP scripting code.
  • PHP statements end with a semi-colon ( ; )
  • Keywords like if, else, while, echo, etc., and classes, functions, and user-defined functions are not case-sensitive.
  • All variable names are case-sensitive.

Using echo/print

In PHP, echo is a commonly used statement. However, we have an alternative on inputting data--print statements. Does it really make a difference whether you use the 'echo' or the 'print' command? W3schools claims they are interchangeable, given that both are employed in the same way: to display the information. The primary distinction between print and echo is that print acts as a function while echo does not. Statements made with print has a return value of 1 while echo statements no return value. The former can be used in expressions. Both can be used to display the value of either a variable or string.

Variables

Variables are symbols or names that represent values. Information such as text strings, numbers, and other numerical or numeric values can all be stored in variables. During the execution of a script, the values of variables may undergo modifications. Some essential information concerning variables:

  • Declaring a variable before assigning a value to it is not required in PHP.
  • Depending on the value of the variable, PHP will automatically transform it to the appropriate data type.
  • A variable can be used in multiple places once it has been declared.
  • Assigning a value to a variable is done through using equal sign (=).

In languages like C, C++, and Java, the coder must first define the variable's name and type. In PHP, types are associated with values rather than variables, hence declaring a variable's type is not necessary before using it. In this way, we have complete freedom to transform the data type associated with a given variable.

In naming variables, there are some conventions that we have to follow, namely:

  • Its structure is $<name of variable>
  • It can have a short name ($i, $x, $v, etc.) or a descriptive name ($counter, $number, $var_score).
  • It can NOT start wth a number. Conversely, it can start with an underscore character ($_feecount).
  • Only letters, numbers, and underscore characters are recognized. Other special characters such as ! ? or * are invalid.
  • Variables are case-sensitive
  • Variables should not contain spaces. You may use the underscore (_) to replicate a space.
